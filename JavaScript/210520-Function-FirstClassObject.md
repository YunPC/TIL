# 함수와 일급 객체

1. 일급 객체

객체 중 다음 조건을 만족하면 일급 객체라 부른다.

1. 런타임에 생성이 가능하다(무명의 리터럴).
2. 변수나 자료구조에 저장할 수 있다.
3. 인수로 사용할 수 있다.
4. 반환값으로 사용할 수 있다.

함수는 일급 객체이므로 객체와 동일하게 사용할 수 있다. 함수는 값으로 평가될 수 있으며, 런타임에 객체를 생성한다.

함수의 인수나 반환값으로 또 다른 함수를 사용할 수 있는 것은 장점이다. 왜냐하면 자바스크립트에서 함수형 프로그래밍이 가능하도록 하기 때문이다.

다만 차이점으로는 함수는 호출이 가능하지만 일반 객체는 호출할 수 없다.

2. 함수 객체의 프로퍼티

함수는 객체이기 때문에 프로퍼티를 가질 수 있다. 데이터 프로퍼티와 접근자 프로퍼티를 가지고 있는데, 이 프로퍼티들 중 일부는 Object의 프로퍼티를 상속받았다.

- arguments 프로퍼티

arguments객체는 함수 호출 시 전달된 인수들의 정보를 저장한 유사 배열 객체이다. 따라서 숫자로 조회가 가능하며 순회도 가능하다. 함수 내부 스코프에 존재한다.

> arguments 프로퍼티
    함수 객체의 arguments 프로퍼티는 ES3이후로 표준에서 폐지되었다. 따라서 함수 내부에서 arguments객체를 참조하는 것을 추천한다.

자바스크립트는 함수 매개변수보다 더 많은 인수를 넘겨도 에러를 발생시키지 않고 무시한다. 더 적은 인수를 넘기는 경우 나머지 매개변수는 undefined로 초기화된다.

넘어온 인자들은 모두 arguments객체에 프로퍼티로 저장되고 인수 정보를 가지고 있다. 이 때 인수 정보는 배열의 인덱스와 유사하게 순서를 숫자값으로 가지고 있다. arguments 객체의 callee 프로퍼티는 호출되어 함수 자신을 가리키고 length프로퍼티는 arguments객체의 길이를 나타낸다.

> arguments 객체의 Symbol(Symbol.iterator)프로퍼티
arguments 객체의 Symbol 프로퍼티는 arguments객체가 이터러블 하도록 만들기 위해서 있다.

arguments 객체는 가변 인자 함수를 구현할 때 용이하다.

> 유사배열 객체와 이터러블
    ES6부터 이터레이션 프로토콜을 준수하면 이터러블이 될 수 있다. ES6부터 arguments객체는 유사 배열 객체이면서 동시에 이터러블이다.

유사 배열 객체는 배열이 아니므로 배열 메서드를 사용하기 위한 간접 호출이 필요한데, 이 과정이 번거롭다. ES6에서 이 문제를 해결하기 위해 Rest 파라미터를 도입했다.

- length 프로퍼티

매개변수의 개수를 나타낸다.

- name 프로퍼티

함수 객체에서의 name프로퍼티는 함수 이름 값을 나타내며, ES5에서 비표준이었다가 ES6에서 표준이 되었다.

name은 ES5와 ES6에서 동작하는 방식이 다르다. 익명 함수의 경우 ES5에서 name은 빈 문자열을 갖지만 ES6의 경우 함수를 가리키는 식별자를 값으로 갖는다.

- `__proto__` 접근자 프로퍼티

모든 객체는 `[[Prototype]]`이란 내부 슬롯을 가지고 있다. 이 슬롯은 프로토타입의 객체를 가리킨다.

`__proto__`는 프로토타입 객체에 간접적으로 접근하기 위해 사용하는 접근자 프로퍼티이다.

> hasOwnProperty 메서드
    전달받은 키가 객체가 가지고 있는 고유한 프로퍼티인지 불리언 값으로 알려준다.

- prototype 프로퍼티

constructor만이 소유하는 프로퍼티이다. 이 프로퍼티는 함수가 객체를 생성하는 생성자 함수로 쓰일 때, 인스턴스가 프로토타입 객체를 가리키도록 한다.
